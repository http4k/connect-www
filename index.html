Functional, modular and testable API adapters

http4k-connect is a set of lightweight API libraries for connecting to popular third-party cloud services and AI backends

See the docs

Modularise API Adapters with the Connect pattern in 4 easy steps:

1. An API Action is just an interface with methods for marshalling HTTP messages

2. Implement any individual API Action with a single class and an extension method

3. An API adapter is just a class with a single function and handles with the transport for the remote API

4. Instantiate the API adapter by passing in the HTTP client and other requirements, and call it as normal

Highlights

Tiny
footprint

In true http4k-style, Connect adapters rely on minimal dependencies and zero reflection, perfect for lightweight and security-conscious applications.

Total
modularisation

Extension of the adapters is as simple as combining function calls or implementing a single data class with just 2 simple methods.

Kotlin-
centric

Every adapter leverages Result types and is exception-safe, so you can finally code like it's Kotlin that you're writing!

Battle-tested
in the wild

Connect adapters have been used to build high volume applications in Banking, Publishing, eCommerce & Government projects.

First class
testability

Most adapters come with an in-memory cloud-service fake which can be used for lightning fast test suites or span up as a server.

All fakes are chaos-enabled, so you can test what happens to your code in failure scenarios.

Total
control

Most API client libraries hide their HTTP client from you, stopping you from implementing observability or customisation.

Connect allows any http4k client module to be plugged in, so you can leverage the full power of http4k

Ultimate
type safety

Adapters leverage TinyTypes to provide typesafe coding and automatic serialisation.

Stop relying on unsafe, stringly-typed APIs.

Simple Key/value
storage

Ships with a storage abstraction for simple key-value storage, and adapters for popular backends.

Plugin in-memory, S3, JDBC or Redis with a single line of code!
